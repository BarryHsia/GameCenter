# Git 提交工作流（精简版）

## 核心规则

**代码修改必须同时更新 CHANGELOG.md，一次提交**

### 何时更新 CHANGELOG

✅ 需要：Bug修复、新功能、重构、优化、安全修复
❌ 无需：纯文档、格式化、依赖更新

### 提交流程

1. 修改代码
2. 更新 `docs/CHANGELOG.md`（在 `---` 后添加条目）
3. 使用 `mcp_github_push_files` 同时推送

## CHANGELOG 格式

**条目格式**: `## YYYY-MM-DD HH:MM | [本次提交] | 类型：描述`

**类型前缀**:
- `新增:` 新功能
- `修复:` Bug修复
- `重构:` 代码重构
- `优化:` 性能优化
- `安全:` 安全修复
- `测试:` 测试相关
- `文档:` 重要文档

**示例**:
```markdown
## 2026-02-11 19:45 | [本次提交] | 修复：解决登录崩溃问题

## 2026-02-11 18:30 | [本次提交] | 新增：用户头像上传功能
```

**规则**:
- 时间倒序（最新在上）
- 描述不超过30字
- 条目间空一行

## Commit 信息规范

- `feat:` 新功能
- `fix:` Bug修复
- `refactor:` 重构
- `perf:` 性能优化
- `docs:` 文档
- `test:` 测试
- `chore:` 构建/工具
